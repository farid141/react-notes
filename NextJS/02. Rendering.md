# Rendering

Sejak Next.js 13 (App Router), React memperkenalkan konsep baru:

- Server Component → dijalankan di server.
- Client Component → dijalankan di browser.

Secara default, semua komponen di Next.js App Router adalah Server Component.
Artinya:

- Tidak ada akses ke window, document, atau localStorage.
- Tidak bisa pakai hook seperti useState, useEffect, useRef.
- Tidak bisa menangani event (onClick, onChange, dsb).

## Client Component

Untuk mendapatkan fungsionalitas reactjs pada umumnya, cukup tambahkan `'use client'` di awal file komponen. Semua child/komponen yang dipakai didalamnya akan otomatis menjadi client component juga. Berikut adalah contohnya.

### Render di Client

Browser akan menerima script js

```jsx
"use client";

import { useState } from "react";

export default function Counter() {
  const [count, setCount] = useState(0);
  return (
    <button onClick={() => setCount(count + 1)}>
      Count: {count}
    </button>
  );
}
```

### Server

Browser akan menerima html (berupa element `main` dan `h1`) dilanjutkan dengan script js

```jsx
// app/page.tsx (server component)
import Counter from "../components/Counter"; // Counter pakai "use client"

export default function Page() {
  return (
    <main>
      <h1>Halaman Utama</h1>
      <Counter /> {/* akan dirender di browser */}
    </main>
  );
}
```
